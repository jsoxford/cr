What's it about?

Today is about shooting for code perfection, not results & output.
  • 5 sessions
  • Same exercise, different constraints
  • Pair programming
  • Retrospectives at end of each session
  • It's a day for you, constraints are just a guide
  1. What is Code Retreat - not Getting Things Done
  2. Not really about perfection - trying different approaches
  3. Constraints are there to help you think about things in different ways.
  4. What you're going to get out of it - TDD/Agile/Pair Programming
  5. You're in control. It's about making the most of the time/space

The rules

  1. Delete your code at the end of each session

Game of Life

  1. Live cell && live neighbours <2 -> die
  2.  Live cell && live neighbours == [2, 3] -> come alive
  3.  Live cell && live neighbours >3 -> die 
  4. Dead cell && live neighbours == 3 -> come alive

  1. Zero-player turn-based game
  2. Infinite board
  3. Each cell alive or dead
  4. Set initial board state
  5. Rules apply to generation of next round
